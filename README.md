I treat the database as a primary test surface, auditing both the data itself and the engine's health.

* **Performance:** Identifying bottlenecks, long-running queries, and deadlock risks.

### 3. Comprehensive Bug Reporting (`/Bug-Reports`)
Clear communication is the bridge between Testing and Development. My reports are designed to reduce "back-and-forth" and accelerate fixes.
* **Structure:** Detailed bug report with environment, steps, and results.
* **Priority:** Risk-based assessment to help Product Owners prioritize the backlog.
